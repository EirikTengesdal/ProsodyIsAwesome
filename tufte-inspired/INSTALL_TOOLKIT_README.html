Abstract

This guide provides cross-platform installation instructions for the Wavelet Prosody Toolkit on Windows, macOS, and Linux. Two installation methods are provided: a global user-level installation for general use across multiple projects, and a per-project installation with isolated dependencies for reproducible research environments. The guide is based on suggestions generated by GitHub Copilot.

Introduction

The Wavelet Prosody Toolkit is a powerful tool for analysing prosodic features in speech. This guide helps you install it on Windows, macOS, or Linux, but no guarantees are made.

Support

For issues specific to this installation guide, please email Eirik Tengesdal at eirik.tengesdal@oslomet.no.

Which Installation Method?

Global Installation

This is the recommended method for general use. Choose this if you:

  • Want to use the toolkit across multiple projects
  • Prefer simple access from anywhere
  • Do not need different toolkit versions for different projects

Per-Project Installation

Choose this if you:

  • Want isolated dependencies per project
  • Need different toolkit versions for different projects
  • Prefer strict reproducibility

Recommendation

For most users, the global installation is simpler and more convenient. Use per-project only when you need strict dependency isolation.

Option 1: Global User-Level Installation

This installs the toolkit once to your user Python environment, making it available from any directory.

Installation

Default Locations

  • Windows: %USERPROFILE%\wavelet_prosody_toolkit
  • macOS/Linux: ~/wavelet_prosody_toolkit

Windows (PowerShell)

# Navigate to where you downloaded the script
cd path\to\scripts

# Install to default location
.\install_toolkit_standalone.ps1

# OR install to custom location
.\install_toolkit_standalone.ps1 -InstallDir "C:\Tools\wavelet_prosody_toolkit"

macOS/Linux (Bash)

# Make script executable
chmod +x install_toolkit_standalone.sh

# Install to default location
./install_toolkit_standalone.sh

# OR install to custom location
./install_toolkit_standalone.sh /path/to/your/preferred/location

Usage

After installation, you can use the toolkit from any directory:

Windows

# Launch the GUI
python -m wavelet_prosody_toolkit.wavelet_gui

# Or use in your Python scripts
python your_analysis_script.py

macOS/Linux

# Launch the GUI
python3 -m wavelet_prosody_toolkit.wavelet_gui

# Or use in your Python scripts
python3 your_analysis_script.py

Python Usage

In your Python code (all platforms):

import wavelet_prosody_toolkit
# Use toolkit functions

Updating the Toolkit

Windows

cd $HOME\wavelet_prosody_toolkit
git pull

macOS/Linux

cd ~/wavelet_prosody_toolkit
git pull

Option 2: Per-Project Installation

This creates a project-specific installation with its own virtual environment.

Installation

Windows (PowerShell)

# In your project directory
.\install_toolkit_project.ps1

macOS/Linux (Bash)

# In your project directory
chmod +x install_toolkit_project.sh
./install_toolkit_project.sh

Usage

Virtual Environment

The toolkit is installed in an isolated environment (.venv) within your project directory.

Windows

# Activate the virtual environment
.\.venv\Scripts\Activate.ps1

# Launch the GUI
python -m wavelet_prosody_toolkit.wavelet_gui

# Run your analysis scripts
python your_analysis_script.py

macOS/Linux

# Activate the virtual environment
source .venv/bin/activate

# Launch the GUI
python -m wavelet_prosody_toolkit.wavelet_gui

# Run your analysis scripts
python your_analysis_script.py

Updating the Toolkit

Windows

cd vendor\wavelet_prosody_toolkit
git pull

macOS/Linux

cd vendor/wavelet_prosody_toolkit
git pull

Prerequisites

Before running either script, ensure you have the following installed:

All Platforms

Python Version

Python 3.8 or later is recommended. The toolkit may work with earlier versions but has not been tested.

Python 3

Windows:

python --version

Download from: python.org/downloads

macOS/Linux:

python3 --version

Git

Windows:

git --version

Download from: git-scm.com/download/win

macOS/Linux:

git --version

Platform-Specific

macOS Only

Xcode Command Line Tools:

xcode-select --install

Windows Only

Microsoft C++ Build Tools (if PyQt6 fails to install):

  • Download Visual Studio Build Tools
  • Select “Desktop development with C++”

Dependencies

The following packages will be automatically installed:

  • PyWavelets
  • scipy
  • numpy
  • matplotlib
  • PyQt6

Quick Start Example

Global Installation

Windows

# Install once
.\install_toolkit_standalone.ps1

# Navigate to your project
cd C:\Users\YourName\my_prosody_analysis

# Create your analysis script
@"
import wavelet_prosody_toolkit as wpt
# Your analysis code here
"@ | Out-File -Encoding utf8 analyze.py

# Run it
python analyze.py

macOS/Linux

# Install once
./install_toolkit_standalone.sh

# Navigate to your project
cd ~/my_prosody_analysis

# Create your analysis script
cat > analyze.py << 'EOF'
import wavelet_prosody_toolkit as wpt
# Your analysis code here
EOF

# Run it
python3 analyze.py

Per-Project Installation

Windows

# In your project directory
.\install_toolkit_project.ps1

# Activate environment
.\.venv\Scripts\Activate.ps1

# Create and run your analysis
python analyze.py

macOS/Linux

# In your project directory
./install_toolkit_project.sh

# Activate environment
source .venv/bin/activate

# Create and run your analysis
python analyze.py

Troubleshooting

Windows Issues

PowerShell execution policy error

Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser

Common Issues

Most installation problems stem from missing prerequisites or incorrect Python/Git installation.

“python: command not found”

  • Ensure Python is in your PATH
  • Try using py instead of python
  • Reinstall Python and check “Add Python to PATH”

PyQt6 installation fails

Install Microsoft C++ Build Tools:

  • Download from: visualstudio.microsoft.com
  • Select “Desktop development with C++”

macOS/Linux Issues

“Permission denied” when running scripts

chmod +x install_toolkit_standalone.sh
chmod +x install_toolkit_project.sh

Python command not found

Replace python3 with python if your system uses that command.

PyQt6 installation fails on macOS

You may need to install additional system libraries:

brew install python-tk

All Platforms

Git clone fails

Ensure you have internet connection and Git is installed:

Windows:

git --version

macOS/Linux:

git --version

Import errors after installation

Ensure you’re using the correct Python environment:

  • For global install: Use system Python
  • For project install: Activate the virtual environment first

Recommended Approach

For most users: Use the global installation:

  • Windows: install_toolkit_standalone.ps1
  • macOS/Linux: install_toolkit_standalone.sh

This is simpler and allows you to use the toolkit across all your projects without reinstalling.

For advanced users: Use per-project installation:

  • Windows: install_toolkit_project.ps1
  • macOS/Linux: install_toolkit_project.sh

Use this when you need strict dependency management or are collaborating with others who need reproducible environments.
